About Engenuity

Engenuity is a 2d/3d game engine made using the Light Weight Java Game Library (LWJGL), with inspiration from the frameworks used for the Unity and Source game engines. The engine started out as a way to condense and streamline utilities for several other projects I was working on. It's roots can be traced to an early project that used Processing's core library. While processing is useful for quick prototypes, the render pipeline (built on Java's AWT) wasn't able to support several of my needs. So I replaced Processing with LWJGL and its OpenGL rendering pipeline.

Projects & Examples Using Engenuity

  • Cubic Ray Marching

    Cubic ray marching is a variation of classic ray marching, but using rectilinear distance, instead of classic euclidean distance. This means the distance a ray can travel in each step is a cube, instead of a sphere. In theory, cubic ray marching can be more performative than classic ray marching. This is because cubic ray marching can avoid the costly square root function, and converges on intersections (specifically with flat planes) faster.

    The implementation made in Engenuity uses the shader and window tools, with the actual marching cubes algorithm being written in GLSL.

  • Dueling Dungeons

    Dueling Dungeons has been developed along side Engenuity, with most of the modular entities, multiplayer and tile map systems being developed specifically for Dueling Dungeons.

  • Pictile

    Pictile uses Engenuity for its windows, audio, and mod loading support.
